# Project Overview｜项目总览

## 项目名称

DataForge Academy｜数据工坊学院

## 项目定位

一个面向数据工程、ETL、大数据、实时数仓、OLAP 分析、数据产品与 AI 数据应用方向的开源项目制学习体系。

## 为什么做这个项目

传统学习路径容易出现三个问题：

1. 知识点割裂：Python、SQL、数据库、Flink、StarRocks 分别学，但不知道如何串成项目。
2. 缺少真实产出：看完课程后没有 GitHub 作品集、项目文档和面试表达。
3. 学习和职业脱节：不知道岗位 JD 要求如何映射到具体学习任务。

DataForge Academy 的目标是用一个长期项目解决这些问题：

- 用项目驱动学习。
- 用岗位能力反推知识点。
- 用每日输出沉淀作品集。
- 用录制过程形成课程资产。
- 用项目文档支持求职和面试。

## 项目主线

主线项目为：面向电商业务的端到端数据平台。

它会逐步覆盖：

```text
数据生成 → 业务库 → ETL → 数据质量 → 数仓建模 → 实时链路 → OLAP 分析 → 查询优化 → 数据看板 → AI 报告 → 求职包装
```

## 学习者路径

不同学习者可以选择不同路径：

- 基础薄弱：先走 Data Foundation。
- 想做 ETL：重点学习 Python、SQL、ETL、调度、数据质量。
- 想做数仓：重点学习 ODS/DWD/DWS/ADS、指标体系、维度建模。
- 想做大数据：重点学习 Kafka、Flink、Spark、Hive。
- 想做实时数仓：重点学习 Flink CDC、Kafka、StarRocks/Doris。
- 想做数据产品：重点学习看板、指标解释、AI 自动报告。
- 想求职：重点学习项目包装、简历描述和面试题。

## 输出标准

每个项目模块都应该包含：

1. 背景说明。
2. 学习目标。
3. 核心知识点。
4. 实操步骤。
5. 代码或 SQL。
6. 常见问题。
7. 面试可讲点。
8. 延伸学习方向。
